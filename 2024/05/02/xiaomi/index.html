5月2日 四大组件详解及避坑指南

四大组件：Activity，Service，BroadcastReceiver，Content Provider

  1. Activity：用于提供用户界面和交互。Activity代表了一个单独的屏幕，用户可在其中执行操作。
  2. Service：后台运行组件，执行长时间运行的任务或处理与用户界面无关的操作
  3. Broadcast Receiver（广播接收器）：接受和响应广播消息
  4. Content Provider（内容提供者）：管理应用程序数据的组件。允许应用程序共享数据给其他应用程序，并提供一种标准化接口。

二、Service

Service是一种后台运行的组件，用于执行长时间运行的任务或处理与用户界面无关的操作。Service可以在后台运行，即使用户切换到其他应用或锁定屏幕，它礽贪可以运行

Service类型：

前台服务和后台服务

Activity弊端：

  1. 复杂业务代码量巨大，Activity愈发臃肿，难以维护
  2. 页面模块难以复用，不适应大屏设备，开发效率低
  3. 启动Activity消耗资源多
  4. 视觉效果也不好

Fragment依附于Activity

Fragment平板适配，引入Fragment可以将屏幕空间充分利用起来

单Activity+多Fragment：性能高，速度最快

多Activity+多Fragment：速度快，更易维护

  1. 静态添加Fragment
  2. 动态添加：使用更加灵活，可通过Bundle设置初始参数

5月5日 Android布局详解与优化

一、布局简介

布局是一种可用于放置很多空间的容器，可以按照一定规律调整内部控件的位置

布局类型：

  1. 传统布局
    1. 框架布局 FrameLayout：常用于包含单一子视图的情况，或者需要子视图层叠的情况
    2. 线性布局 LinearLayout：以线性排列子视图，可以设置水平或垂直排列
    3. 相对布局 RelativeLayout：相对于其他布局
      1. l
    4. 表格布局 RableLayout
    5. 绝对布局 AbsoluteLayout
  2. 新型布局
    1. 约束布局 ConstrainLayout
      1. 功能强大，能几乎实现其他布局所有的功能
      2. 能减少布局层次的嵌套，有性能的优势
      3. 可视化操作的增强，大部分界面是有约束布局都可以通过可视化编辑区完成

共有属性

  1. gravity

    1. layout_gravity：控件相对于父控件的对齐方式
    2. gravity：控件内部的对齐方式，center,top|center,

    dp是根据手机屏幕调整，sp根据手机文字调整，textsize一般用sp

  2. padding内边距

    1. paddingTop：上间距
    2. paddingBottom：下间距
    3. paddingLeft：左间距
    4. paddingRight：右间距

  3. width和height

    1. layout_height：控件的高度
    2. layout_width：控件的宽度

  4. margin

    1. layout_marginLeft 控件的左间距
    2. layout_marginRight 控件的右间距
    3. layout_marginTop 控件的上间距
    4. layout_marginBottom 控件的下间距

一、组件库的依赖，定制和发布

定义：一组可重用的UI组件和工具

内容：UI组件和工具：按钮、文本框、图像视图等，以及用于处理应用程序逻辑的工具，如活动、服务、广播接收器等

优势：

  1. 提高开发效率，降低开发成本

  3. 提高应用程序的可扩展性

组件库文件格式
